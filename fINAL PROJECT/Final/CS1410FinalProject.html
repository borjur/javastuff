Problem

Create a contact management system.

Specification

  • Create super class Contact with subclasses BusinessContact, FamilyContact and FriendContact
  • Define appropriate fields and methods for each class.
  • Take advantage of polymorphic method behavior when searching, displaying, reading/writing the contact list.
  • Use regular expressions to valid the format for phoneNumber, emailAddress and other fields as appropriate.
  • Implement the exception handling mechanism with custom exception classes.
  • Implement unit testing using JUnit:
    • Create a unit test for each class composed of several test cases each designed to test the response of a single method to a specific set of inputs.
    • Create a test suite that is a collection of all unit tests.
    • Use a test runner (JUnit) to run the tests and report results.

    NOTEs:

    1. View this PPT for a quick overview of unit testing.
    2. Download Subscription.zip for an example implementation.
    3. Click here to view a series of Eclipse tutorials that implement unit testing.

  • Store the records in memory in an ArrayList.
  • Store the records on disk in the binary file contacts.ser.
  • Automatically read the records from disk into memory when the program runs.
  • Automatically write the records to disk when the program ends.
  • Create an appropriate GUI that allows for operations such as:
    • Add
    • Delete
    • Search
    • Display
  • Use the javadoc utility to create documentation for all the .java files in your project. Include a feature in the GUI that displays the documentation.
  • Make sure your source code conforms to the CS Department Style Guide, to include method header comments.
